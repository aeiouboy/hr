# Recruitment & Onboarding (การสรรหาและการรับพนักงานใหม่)

เอกสารนี้อธิบาย module ที่ครอบคลุมวงจรชีวิตพนักงานตั้งแต่การสมัครงานจนถึงการลาออก

---

## การสรรหาพนักงาน (`recruitment`)

**หน้าที่หลัก:** ระบบประกาศตำแหน่งงานภายในองค์กร ให้พนักงานสามารถค้นหาและสมัครงานได้ด้วยตนเอง และให้ HR สามารถจัดการใบสมัครและติดตาม pipeline การสรรหาได้ในที่เดียว

**ใครใช้ได้บ้าง:** Employee, Manager, HR Admin, HR Manager

**สิ่งที่ทำได้:**

*สำหรับพนักงานทั่วไป (Employee):*
- ดูกระดานประกาศงานภายใน (Internal Job Board) และกรองตำแหน่งตามสถานที่ทำงาน / แผนก / ประเภทงาน
- ค้นหาตำแหน่งงานด้วยชื่อตำแหน่งหรือชื่อแผนก (รองรับทั้งภาษาไทยและอังกฤษ)
- ดูรายละเอียดตำแหน่งงาน เช่น คำอธิบายงาน คุณสมบัติที่ต้องการ สวัสดิการ และช่วงเงินเดือน
- ดาวน์โหลดไฟล์ Job Description (JD) แนบท้าย
- กรอกใบสมัครออนไลน์ พร้อมแนบ CV/Resume (รองรับ PDF, DOC, DOCX ขนาดไม่เกิน 5MB)
- ติดตามสถานะใบสมัครที่ตัวเองยื่นไว้ พร้อมดู Timeline การเปลี่ยนแปลงสถานะ
- ดูตารางนัดสัมภาษณ์ (วัน เวลา สถานที่ ประเภทสัมภาษณ์)

*สำหรับ HR Admin / HR Manager (ต้องมีสิทธิ์ `manage_recruitment`):*
- ดู Dashboard สรุปภาพรวมการสรรหา ได้แก่ จำนวนตำแหน่งที่เปิดรับ, ใบสมัครทั้งหมด, ใบสมัครใหม่, อยู่ระหว่างสัมภาษณ์ และ Offer ที่รอการตอบรับ
- ดูรายการใบสมัครทั้งหมดในรูปแบบตาราง พร้อมข้อมูลผู้สมัคร ตำแหน่ง วันที่สมัคร และแหล่งที่มา
- ดูรายละเอียดผู้สมัคร เช่น ประสบการณ์ทำงาน ทักษะ จดหมายสมัครงาน และโน้ตภายใน
- อัปเดตสถานะใบสมัคร (ใหม่ / Shortlist / สัมภาษณ์ / เสนองาน / รับเข้า / ปฏิเสธ)
- ดาวน์โหลด CV ของผู้สมัคร
- ระบุว่าผู้สมัครเป็นพนักงานภายใน (Internal Candidate) หรือไม่

**ข้อมูลที่แสดง:**
- การ์ดตำแหน่งงานแสดง: ชื่อตำแหน่ง, บริษัท, สถานที่, แผนก, ประเภทการจ้าง, ช่วงเงินเดือน, จำนวน Headcount ที่เปิดรับ, วันปิดรับสมัคร, และป้าย "เร่งด่วน" เมื่อเหลือเวลาน้อยกว่า 7 วัน
- ใบสมัครแสดง: ชื่อผู้สมัคร, อีเมล, เบอร์โทร, วันที่สมัคร, แหล่งที่มา (LinkedIn / JobsDB / เว็บบริษัท / Referral ฯลฯ), สถานะปัจจุบัน
- Timeline สถานะใบสมัคร แสดงประวัติการเปลี่ยนแปลงพร้อมวันที่และผู้อัปเดต
- ตารางนัดสัมภาษณ์แสดงประเภท (Technical / HR / Final), วันเวลา, สถานที่, และสถานะ (นัดแล้ว / เสร็จสิ้น)

**หมายเหตุ:** ตำแหน่งงานที่แสดงในกระดาน Job Board จะแสดงเฉพาะตำแหน่งที่มีสถานะ "เปิดรับ" (`open`) และเป็นการประกาศภายใน (`isInternal`) เท่านั้น แท็บ "จัดการการสรรหา" จะแสดงเฉพาะผู้ที่มีสิทธิ์ `manage_recruitment` ซึ่งได้แก่ HR Admin ขึ้นไป

## การคัดกรองผู้สมัคร (`candidate-screening`)

**หน้าที่หลัก:** ระบบสนับสนุน HR และ Hiring Manager ในการคัดกรองผู้สมัครงาน ตั้งแต่การให้คะแนน Match Score อัตโนมัติ นัดสัมภาษณ์ ประเมินผล เปรียบเทียบผู้สมัคร ไปจนถึงการออก Offer Letter

**ใครใช้ได้บ้าย:** HR Admin, HR Manager (เป็นหน้าจอสำหรับ HR โดยเฉพาะ)

**สิ่งที่ทำได้:**

*แท็บ Screening (การคัดกรองเบื้องต้น):*
- ดูรายชื่อผู้สมัครทั้งหมดในรูปแบบตาราง พร้อม Match Score แสดงเป็น % และแถบสีแดง/เหลือง/เขียว
- กรองผู้สมัครตามสถานะ (กำลังคัดกรอง / Shortlist / นัดสัมภาษณ์ / ประเมินแล้ว / ส่ง Offer / ปฏิเสธ)
- กรองตามตำแหน่งที่สมัคร
- ค้นหาผู้สมัครตามชื่อหรืออีเมล
- เลือกผู้สมัครหลายคนพร้อมกัน (Bulk Select) เพื่อทำ Shortlist, ปฏิเสธ หรือนัดสัมภาษณ์พร้อมกัน
- ดูรายละเอียดผู้สมัครในหน้าต่าง Modal แสดงประวัติการศึกษา ประสบการณ์ ทักษะ และผลการคัดกรองอัตโนมัติ (คะแนนแยกตามด้านการศึกษา ประสบการณ์ ทักษะ และภาษา)

*แท็บ Interviews (การนัดสัมภาษณ์):*
- นัดสัมภาษณ์พร้อมกำหนดประเภท (Technical / HR / Final ฯลฯ) รอบที่ วันเวลา ห้อง และรายชื่อ Panelist
- เลือกห้องประชุมจากรายการห้องที่มีในระบบ (แสดงชั้น ความจุ และอุปกรณ์)
- ดูตาราง Calendar สัมภาษณ์รายสัปดาห์ของผู้สมัคร
- อัปเดตสถานะสัมภาษณ์ (นัดแล้ว / เสร็จสิ้น / ยกเลิก)

*แท็บ Evaluation (การประเมินผลสัมภาษณ์):*
- บันทึกผลการประเมินหลังสัมภาษณ์ แยกคะแนนเป็น 5 ด้าน: Technical Skills, Communication, Problem Solving, Culture Fit, และ Overall Rating
- บันทึกข้อสังเกตและบันทึกโน้ตจากผู้สัมภาษณ์
- ระบุคำแนะนำสุดท้าย (รับเข้า / รับแบบมีเงื่อนไข / ปฏิเสธ)

*แท็บ Comparison (การเปรียบเทียบผู้สมัคร):*
- เลือกผู้สมัครหลายคนเพื่อเปรียบเทียบแบบ Side-by-Side ดูข้อมูลคู่ขนาน เช่น ประสบการณ์ การศึกษา ทักษะ เงินเดือนปัจจุบัน/ที่คาดหวัง และ Match Score
- ช่วยสนับสนุนการตัดสินใจเลือกผู้สมัครที่เหมาะสมที่สุด

*แท็บ Offers (การออก Offer Letter):*
- ดู Offer ที่ส่งออกไปแล้ว พร้อมสถานะการตอบรับ (รอตอบ / ยอมรับ / ปฏิเสธ)
- ดูประวัติการอนุมัติ Offer แสดงรายชื่อผู้อนุมัติในแต่ละขั้นตอน (Hiring Manager / HR Director / Finance)
- ดูรายละเอียด Offer ได้แก่ ตำแหน่ง เงินเดือนที่เสนอ วันเริ่มงาน วันหมดอายุ Offer และสวัสดิการ

**ข้อมูลที่แสดง:**
- Match Score แสดงเป็น % ประกอบด้วยคะแนนย่อย 4 ด้าน (การศึกษา ประสบการณ์ ทักษะ ภาษา)
- สถานะ Pipeline ของผู้สมัครแต่ละคน ตั้งแต่ screening → shortlisted → interview_scheduled → evaluated → offer_sent → hired
- ข้อมูลเงินเดือนปัจจุบันและที่คาดหวังของผู้สมัคร (สำหรับ HR เท่านั้น)
- จำนวน Headcount ที่เปิดรับในแต่ละตำแหน่ง

**หมายเหตุ:** การออก Offer ต้องผ่านการอนุมัติหลายขั้นตอน (Multi-level Approval) ได้แก่ Hiring Manager, HR Director, และ Finance ก่อนส่งถึงผู้สมัคร

## การปฐมนิเทศพนักงานใหม่ (`onboarding`)

**หน้าที่หลัก:** ระบบบริหารจัดการกระบวนการรับพนักงานใหม่ตั้งแต่ก่อนเริ่มงาน ผ่านวันแรกของการทำงาน ไปจนถึงการสิ้นสุดทดลองงาน โดยมี Checklist และการติดตามความคืบหน้าในแต่ละขั้นตอน

**ใครใช้ได้บ้าง:** HR Admin, HR Manager (เป็น module สำหรับทีม HR บริหารจัดการกระบวนการ)

**สิ่งที่ทำได้:**

*แท็บ Pre-boarding (ก่อนวันเริ่มงาน):*
- จัดการ Checklist งานที่ต้องเตรียมก่อนพนักงานใหม่มารายงานตัว จัดกลุ่มตามหมวดหมู่
- มาร์ค Checklist รายการว่า "เสร็จสิ้น", "กำลังดำเนินการ", "รอดำเนินการ" หรือ "ไม่เกี่ยวข้อง"
- ดูวันที่นัดหมาย (เช่น วันนัดตรวจสุขภาพ) และโรงพยาบาลที่นัด
- ดูวันที่คาดว่าจะเสร็จสำหรับรายการที่กำลังดำเนินการ

*แท็บ Day One (วันแรกของการทำงาน):*
- ดูตารางกิจกรรมวันแรก พร้อมเวลาที่ประมาณไว้ และระยะเวลาของแต่ละกิจกรรม
- เห็นว่าใครรับผิดชอบแต่ละกิจกรรม (ผู้ดูแล)
- มาร์ครายการในตาราง Day One ว่าเสร็จสิ้นหรือยังรอดำเนินการ

*แท็บ Employee ID (การออกรหัสพนักงาน):*
- ออกรหัสพนักงานใหม่ตามรูปแบบที่กำหนดในระบบ
- ยืนยันรหัสพนักงาน (Confirm ID)
- ส่งคำขอพิมพ์บัตรพนักงาน (Request ID Card Printing)
- สร้าง User Account ในระบบให้พนักงานใหม่

*แท็บ Orientation (การปฐมนิเทศ):*
- จัดการโปรแกรมปฐมนิเทศ แยกตามหมวดหมู่ (เช่น ด้านกฎระเบียบบริษัท ด้านเทคนิค ด้านวัฒนธรรมองค์กร)
- รองรับกิจกรรมหลายประเภท: E-Learning, ห้องเรียน (Classroom), การประชุมแบบตัวต่อตัว, เอกสาร, และลายเซ็นต์
- ดูวันที่และเวลาของกิจกรรมที่นัดไว้ พร้อมสถานที่และรายชื่อผู้เกี่ยวข้อง
- เลือกและมอบหมาย Mentor ให้พนักงานใหม่จากรายชื่อที่มีในระบบ (แสดงข้อมูล Mentor เช่น อายุงาน จำนวน Mentee ปัจจุบัน คะแนนจาก Mentee และทักษะ)

*แท็บ Probation (การทดลองงาน):*
- ดูข้อมูลการทดลองงาน: วันเริ่ม วันสิ้นสุด ระยะเวลา (จำนวนวัน) และสถานะ
- ดูตารางการประเมินระหว่างทดลองงาน พร้อมวันครบกำหนดและสถานะ (รอดำเนินการ / เสร็จสิ้น)
- เพิ่มและติดตาม Goal การทดลองงาน กำหนดน้ำหนัก (%) และดู Progress Bar ของแต่ละ Goal
- ดูและจัดการการแจ้งเตือน (Reminder) ที่กำหนดไว้ล่วงหน้า พร้อมสถานะว่าส่งแล้วหรือยังรอ

**ข้อมูลที่แสดง:**
- Summary Card แสดงข้อมูลพนักงานใหม่: ชื่อ ตำแหน่ง แผนก วันเริ่มงาน สถานะปัจจุบัน และชื่อ HR Coordinator
- Progress Overview แสดงเปอร์เซ็นต์ความคืบหน้าของแต่ละขั้นตอน (Pre-boarding / Day One / Orientation / Probation) ในรูปแบบ Circular Progress Bar

**หมายเหตุ:** รายการ Checklist ที่มีเครื่องหมาย `*` หมายถึงรายการบังคับ (Required) ที่ต้องดำเนินการให้เสร็จ รายการที่ถูกมาร์คว่า "ไม่เกี่ยวข้อง" (`not_applicable`) จะถูกนับว่าเสร็จสิ้นแล้วในการคำนวณ Progress

## การลาออก (`resignation`)

**หน้าที่หลัก:** ระบบบันทึกและจัดการกระบวนการลาออกตั้งแต่การยื่นใบลาออก การตรวจรับส่งงาน (Clearance) ไปจนถึงการสรุปการชำระเงินสุดท้าย (Final Settlement)

**ใครใช้ได้บ้าง:** Employee (สำหรับยื่นใบลาออกของตนเอง), HR Admin, HR Manager (สำหรับจัดการกระบวนการ)

**สิ่งที่ทำได้:**

*แท็บ บันทึกการลาออก (Recording):*
- กรอกและยื่นใบลาออกออนไลน์ ระบุ: วันที่ลาออก, วันทำงานวันสุดท้าย, เหตุผลประเภท (เช่น งานใหม่ / เกษียณ / เหตุส่วนตัว ฯลฯ) และรายละเอียดเพิ่มเติม
- ระบบคำนวณระยะเวลาแจ้งล่วงหน้า (Notice Period) อัตโนมัติจากวันที่ที่กรอก
- กำหนดนัดหมาย Exit Interview พร้อมเลือกวันที่
- ดูสถานะใบลาออกที่ยื่นไว้แล้ว (Draft / รออนุมัติ / อนุมัติแล้ว / รอ Clearance / Clearance เสร็จสิ้น / Final Settlement / เสร็จสมบูรณ์ / ยกเลิก)
- ดูสรุปความคืบหน้าของ Clearance Checklist (เสร็จแล้ว / กำลังดำเนินการ / รอดำเนินการ) พร้อม Progress Bar เป็น %

*แท็บ Clearance Checklist (การตรวจรับส่งงาน):*
- ดู Checklist การส่งมอบงานจัดกลุ่มตามหมวดหมู่ เช่น ทรัพย์สินบริษัท, ระบบ IT, เอกสาร, การเงิน ฯลฯ
- แต่ละรายการแสดง: ชื่อรายการ ผู้รับผิดชอบ (พนักงาน / HR / IT / Finance ฯลฯ) สถานะ และหมายเหตุ
- อัปเดตสถานะรายการ Clearance (รอดำเนินการ / กำลังดำเนินการ / เสร็จสิ้น / ไม่เกี่ยวข้อง) พร้อมบันทึกโน้ต
- ระบบบันทึกวันและเวลาที่ Sign-off พร้อมรหัสผู้อนุมัติ

*แท็บ Final Settlement (การชำระเงินสุดท้าย):*
- ดูรายละเอียดการชำระเงิน แยกเป็น 3 ส่วน:
  - รายรับ: เงินเดือนค้างจ่าย, เงินวันหยุดพักผ่อนที่ไม่ได้ใช้ (Leave Encashment), โบนัส
  - รายหัก: เงินกู้และเงินทดรองจ่าย
  - กองทุนสำรองเลี้ยงชีพ (Provident Fund)
- ดูยอดรวมสุทธิที่พึงได้รับ พร้อมยอดรวมทั้งสิ้นรวม Provident Fund
- ดูรายละเอียดการโอนเงิน: วันที่กำหนดจ่าย และบัญชีธนาคาร
- ดูสถานะ Settlement (รอดำเนินการ / ดำเนินการแล้ว)

**ข้อมูลที่แสดง:**
- Banner สถานะใบลาออก แสดงสีตามสถานะ (เทา / เหลือง / น้ำเงิน / ส้ม / เขียว / ม่วง ฯลฯ) พร้อมไอคอนและวันที่ยื่น
- สรุป Clearance Progress แสดงจำนวนรายการแยกตามสถานะ และ Progress Bar รวม
- ตัวเลข Final Settlement แสดงในสกุลเงินบาท (THB) พร้อม Format จำนวนตัวเลข

**หมายเหตุ:** ใบลาออกที่ยื่นแล้วต้องผ่านการอนุมัติก่อน (สถานะ `pending_approval`) จึงจะเข้าสู่ขั้นตอน Clearance ได้ แท็บ Clearance และ Settlement จะแสดงเฉพาะเมื่อมีใบลาออกในระบบแล้วเท่านั้น หากยังไม่มี ระบบจะแสดงปุ่มให้เริ่มกระบวนการลาออก
