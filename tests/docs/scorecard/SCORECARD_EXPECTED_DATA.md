## Translation Keys That Should NOT Appear

The following should never be visible on the page (they are translation keys):

- `scorecard.cgCompetency`
- `scorecard.noCompetencies`
- `scorecard.personalAssessmentHistory`
- `scorecard.overallRating`
- `scorecard.boxLabels.stars`
- Any string starting with `scorecard.`

If you see these, it means translations are missing or not being applied correctly.

---

## Values That Should NOT Appear

- `undefined`
- `null` (as displayed text)
- `NaN`
- `[object Object]`
- Empty rating badges ( /5 with no number)
- Blank dates
- Missing names

## Color Scheme

### Background Colors
- Purple: #7C3AED (competencies, talent reference)
- Indigo: #4F46E5 (assessment history, final calibration)
- Blue: #2563EB (assessment summary, development objectives)
- Amber: #F59E0B (key successes)
- Teal: #14B8A6 (strengths & development)
- Green: #10B981 (development objectives, strengths)
- Orange: #F97316 (development areas)
- Pink: #EC4899 (9-box matrix)

### Badge Colors
- Green: High ratings (5/5), completed status, high impact
- Blue: Medium-high ratings (4/5), in progress
- Yellow: Medium ratings (3/5), medium impact
- Orange: Low-medium ratings (2/5), development priorities
- Red: Low ratings (1/5), low performance

## Responsive Breakpoints

- **Mobile** (< 640px): Single column, stacked sections
- **Tablet** (640px - 1024px): Some two-column layouts
- **Desktop** (> 1024px): Full two-column layouts where applicable

Key responsive changes:
- Two-column sections (Strengths/Development, Career Aspirations) become single column on mobile
- 9-Box grid scales down but remains 3×3
- Font sizes scale appropriately
- Padding reduces on smaller screens
